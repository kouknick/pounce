As with every other form component, the DateRangeInput is a controlled component and so it
requires for you to specify the `value` and the `onChange` handler.

Any extra prop that you specify will be given to the underlying input element of the module.

---

```typescript jsx
const DateRangeExample = () => {
  const [dates, setDates] = React.useState([]);

  const onChange = React.useCallback(changed => {
    setDates(changed);
  });
  return (
    <Box>
      <Box>
        <DateRangeInput
          withPresets
          labelStart="Start"
          labelEnd="End"
          placeholderStart="Start date"
          placeholderEnd="End date"
          onChange={setDates}
          value={dates}
          id="hello-date-range"
        />
      </Box>
    </Box>
  );
};

<DateRangeExample />;
```

const DateRangeExample = () => {
  const [dates, setDates] = React.useState([new Date(), new Date(Date.now() + 604800)]);

  const onChange = React.useCallback(changed => {

  });
  return (
    <Box>

          <span>

          placeholderEnd="End date"
          onChange={setDates}
          value={dates}
          id="hello-date-range"

      </Box>

  );
};
